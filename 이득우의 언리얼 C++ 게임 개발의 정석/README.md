# 이득우의 언리얼 C++ 게임 개발의 정석
## CHAPTER 2 액터의 설계
- 액터 = 언리얼에서 콘텐츠를 구성하는 최소 단위 물체 -> FB의 엔티티
- 주요 컴포넌트
	- 스태틱메시 : 애니메이션이 없는 모델링 애셋. 시각적 기능, 물리적 기능 제공. 주로 배경 물체에 사용
	- 스켈레탈메시 : 애니메이션 정보가 있는 모델링 애셋. 시각적 기능, 애니메이션, 캐릭터의 물리 기능 제공. 주로 캐릭터에 사용
	- 콜리전 : 물리적 기능 제공
	- 카메라
	- 오디오
	- 파티클 시스템 : 이펙트를 위한 기능
	- 라이트
	- 무브먼트 : 물체에 특정한 움직임을 부여하는 기능
- 액터의 멤버 변수
	- 포인터 변수의 메모리를 자동으로 관리하게 하려면 UPROPERTY 매크로를 사용
- 언리얼 오브젝트 클래스 규칙
	- 클래스 선언 매크로: 클래스 선언 윗줄에 UCLASS 매크로를 사용하고 클래스 내부에는 GENERATED_BODY 매크로를 선언한다.
	- 클래스 이름 접두사: A는 액터클래스에 사용, U는 액터가 아닌 클래스에 사용. 예: 액터는 AFountain, 스태틱메시 컴포넌트는 액터가 아니므로 UStaticMeshComponent라는 클래스 이름이.
	- generated.h 헤더 파일: 언리얼 오브젝트 선언의 마지막 #include 구문에 이 헤더 파일을 반드시 선언해야 한다.
	- 외부 모듈에의 공개 여부: 윈도우의 DLL 시스템은 DLL 내 클래스 정보를 외부에 공개할지 결정하는 _declspec(dllexport)라는 키워드를 제공한다. 언리얼 엔진에서 이 키워드를 사용하려면 '모듈명_API'라는 키워드를 클래스 선언 앞에 추가한다. 이 키워드가 없으면 다른 모듈에서 해당 객체에 접근할 수 없다.
- UPROPERTY
	- VisibleAnywhere, EditAnywhere, Category= 등 키워드가 여러개 있다.
- 로그
	- UE_LOG(Category, Warning, TEXT("A"));
- 어설션
	- check : 크래시 내고 에디터 종료 시킴
	- ensure : 출력 로그에 오류만 띄워줌

## CHAPTER 3 움직이는 액터의 제작
- 액터에 스태틱 메시 컴포넌트, 포인트라이트 컴포넌트, 파티클시스템 컴포넌트 연결
- RotatingMovementComponent로 회전 동작

## CHAPTER 4 게임플레이 프레임워크
- 게임모드
	- 플레이어가 입장할 때마다 플레이어를 점검하고 게임에 입장한 플레이어가 조종할 액터를 생성해 전달
- 폰
	- 플레이어가 조종할 수 있는 액터
- 플레이어의 입장
	- 플레이어가 입장하면 게임모드가 플레이어에게 폰을 배정해준다.
	- 플레이어 컨트롤러라는 액터도 함께 배정
		- 플레이어 컨트롤러: 폰을 조종하는 역할을 맡고, 플레이어가 입장할 때 배정되며 배정된 플레이어 컨트롤러는 변경할 수 없다.
		- 플레이어는 플레이어 컨트롤러를 통해 현재 조종하는 폰을 버리고 다른 폰으로 옮겨가 조종할 수 있다.
	- 플레이어가 게임에 입장하는 것을 언리얼에서는 로그인(Login)이라고 하고, 로그인 과정에서 플레이어에게 할당할 플레이어 컨트롤러를 생성한다.
	- PostLogin 같은 함수를 오버라이드하여 동작시킬 수 있음
